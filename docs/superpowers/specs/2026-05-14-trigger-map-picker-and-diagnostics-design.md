# 位置触发器地图选点与触发器诊断设计

## 背景

vFlow 已有位置触发器，当前通过 `latitude`、`longitude`、`radius`、`location_name` 参数描述地理围栏，并由 `LocationTriggerHandler` 基于 Android `LocationManager` 判断进入或离开事件。编辑器目前支持手动输入坐标和读取当前位置，但缺少可视化地图选点。

用户还希望在编辑触发器时增加测试入口，但明确要求只检查触发条件和权限，不执行工作流后续动作。

## 目标

1. 位置触发器编辑器支持高德地图可视化选点。
2. 选点结果回填现有参数，不引入破坏性工作流格式变更。
3. 触发器编辑器提供“测试触发器”按钮。
4. 测试触发器只做诊断，不执行工作流动作。
5. 诊断结果能说明参数问题、权限缺失、当前环境是否满足触发条件，以及无法诊断的限制。

## 非目标

1. 不把工作流触发判断迁移到高德服务端地理围栏。
2. 不新增用户在 App 内填写高德 Key 的设置项。
3. 不在测试按钮中伪造系统事件并启动真实工作流。
4. 不改变现有触发器参数 ID、触发器 ID、输出 ID 或枚举持久化值。

## 方案概览

推荐方案分为两个独立能力：

1. 高德地图选点作为位置触发器的编辑体验增强。
2. 触发器诊断作为编辑器通用能力，由各触发器按需提供可测试逻辑。

位置触发器仍保存：

- `event`: `enter` 或 `exit`
- `latitude`: 纬度
- `longitude`: 经度
- `radius`: 半径，单位米
- `location_name`: 位置名称

## 高德地图选点设计

### 接入方式

新增高德地图选点 Activity，例如 `AmapLocationPickerActivity`。该页面负责：

1. 展示高德地图。
2. 支持地图点选或拖动中心点。
3. 支持地点搜索或 POI 选择。
4. 显示当前半径范围预览。
5. 返回选中的纬度、经度和地点名称。

`LocationTriggerUIProvider` 新增“地图选点”按钮，通过已有 `onStartActivityForResult` 通道打开该 Activity。返回后更新编辑器中的 `latitude`、`longitude`、`location_name`，并触发 `onParametersChanged()`。

### Key 和构建配置

高德 Android Key 使用构建配置注入：

1. 从本地配置读取，例如 `local.properties` 中的 `AMAP_API_KEY`。
2. 通过 Gradle `manifestPlaceholders` 注入 `AndroidManifest.xml` metadata。
3. 缺少 Key 时仍允许项目构建，但地图选点按钮进入时提示未配置高德 Key。

不把高德 Key 设计为普通用户运行时输入项，因为 Android Key 通常与包名和签名 SHA1 绑定。

### 隐私与权限

高德 SDK 初始化必须遵循官方隐私合规要求，在用户隐私授权之后设置隐私同意状态。位置触发器已有前台和后台定位权限要求；地图选点页只需要前台定位或地图选点能力，后台位置仍由触发器监听负责。

### 兼容性

地图选点只是参数编辑器增强。已有工作流 JSON 不需要迁移；没有高德 Key 或设备不支持地图时，手动输入和“使用当前位置”仍可用。

## 触发器诊断设计

### 用户体验

在触发器编辑底部表单增加“测试触发器”按钮。按钮执行诊断并展示结果：

1. 成功：当前配置和权限满足，当前环境可命中该触发器条件。
2. 未命中：权限和参数正常，但当前环境不满足条件。
3. 缺权限：列出缺失权限，并提供跳转授权入口或现有权限恢复入口。
4. 参数错误：显示具体字段问题，例如坐标非法、半径无效。
5. 不支持诊断：说明该触发器无法在编辑器中可靠检测，例如依赖未来系统广播或外部事件。

诊断按钮不保存编辑器内容，但会先读取当前表单状态，以用户眼前看到的值为准。

### 内部接口

新增轻量诊断接口，例如：

```kotlin
interface TriggerDiagnosticProvider {
    suspend fun diagnose(context: Context, step: ActionStep): TriggerDiagnosticResult
}
```

结果模型表达状态、标题、说明、缺失权限和可选动作。触发器模块可以选择实现或通过注册表提供诊断器；未实现的触发器返回“不支持诊断”。

### 位置触发器诊断

位置触发器诊断步骤：

1. 校验 `event`、`latitude`、`longitude`、`radius`。
2. 检查前台定位和后台定位权限。
3. 获取最近位置或当前可用位置。
4. 计算当前位置与围栏中心距离。
5. 根据 `event` 输出诊断结果：
   - `enter`: 当前在半径内则为可命中，否则未命中。
   - `exit`: 如果已有围栏状态且之前在围栏内、当前在围栏外，则可命中；否则说明离开事件需要历史进入状态。

因为离开事件依赖历史状态，诊断结果必须明确区分“当前不在范围内”和“已满足离开触发的历史条件”。

### 其他触发器诊断

第一阶段只要求位置触发器具备完整条件检查。通用按钮对其他触发器提供基础权限和参数校验；能可靠检查当前状态的触发器可以逐步补充专属诊断器，例如 Wi-Fi、蓝牙、电量、屏幕状态。

## 错误处理

诊断过程中的预期失败以诊断结果返回，不抛出到 UI：

1. 权限缺失返回 `MISSING_PERMISSION`。
2. 参数非法返回 `INVALID_CONFIG`。
3. 当前状态不可获取返回 `UNKNOWN` 或 `UNSUPPORTED`。
4. SDK 初始化失败返回可读错误，不影响手动坐标编辑。

诊断不产生 `ExecutionResult`，也不调用真实工作流执行路径。

## 测试策略

1. 为位置触发器距离和事件诊断添加 JVM 单元测试。
2. 为权限检查和诊断结果映射添加 focused helper tests。
3. 为编辑器参数读取和诊断按钮入口添加 UI model/helper 测试，优先避免重型 instrumentation。
4. 保留现有 `LocationTriggerHandler` 触发逻辑测试边界，不把高德 SDK 纳入 JVM 单元测试。
5. 构建验证需要覆盖 app 编译；设备验证需要确认地图页可打开、选点可回填、诊断不会执行工作流。

## 验收标准

1. 位置触发器编辑器显示“地图选点”入口。
2. 配置高德 Key 后可打开地图选点页，并能回填坐标和地点名称。
3. 未配置高德 Key 时有清晰提示，不影响手动输入和当前位置按钮。
4. 触发器编辑器显示“测试触发器”入口。
5. 点击测试入口不会执行工作流动作，也不会写入日志为真实触发执行。
6. 位置触发器能诊断权限、参数、当前位置是否在围栏内。
7. 离开事件诊断能说明历史状态限制。
8. 现有工作流导入、导出、保存格式保持兼容。

## 实现风险

1. 高德 SDK 版本和 Maven 源配置可能影响构建稳定性，需要固定版本并记录来源。
2. Android 位置权限和后台定位权限在不同系统版本上授权路径不同，诊断 UI 不能假设一次点击即可授权完成。
3. 触发器诊断如果复用真实执行协调器会误执行工作流，必须通过接口边界阻断。
4. 离开地理围栏依赖历史状态，不能简单用“当前位置在范围外”判断为可触发。
